Initialization vector

Some modes for block ciphers like CBC combine a block with the last encrypted block before encryption. To enable this for the first block, the IV is used instead of an encrypted block.

The Initialization Vector (IV) SHOULD be chosen at random, and MUST be unpredictable. [...] For block ciphers, the IV length is of length SecurityParameters.record_iv_length, which is equal to the SecurityParameters.block_size.

TLS 1.2 specification, RFC 5246, p. 23
